01 — Aperçu

Qu'est-ce que GRONDIN v2 ?

GRONDIN Response Handler v2.0 est un outil web de traitement automatisé d'emails destiné aux professionnels de la santé. Il analyse le contenu d'un email entrant, le classe dans une catégorie, génère une réponse adaptée et peut envoyer une alerte via Telegram.

Deux moteurs de classification sont disponibles : mots-clés locaux (aucune connexion externe requise) ou API Claude (Anthropic) pour une classification plus intelligente et contextuelle.

INFO

L'outil fonctionne entièrement dans votre navigateur. Aucune donnée n'est transmise à l'extérieur, sauf si vous activez l'API Claude ou les alertes Telegram.

02 — Catégories

Les 4 types de classification

Chaque email traité est assigné automatiquement à l'une des catégories suivantes :

🟢
Positive Meeting

Demande de rencontre ou confirmation de rendez-vous. L'email exprime une volonté de se rencontrer ou un suivi de réunion.

🟡
Info Requirement

Demande d'information générale. L'expéditeur cherche des renseignements, documents ou clarifications.

🔵
Proof of Status

Demande de preuve de statut ou document officiel : attestation, certificat médical, confirmation administrative.

🔴
Actif QR Other

Catégorie fourre-tout pour les emails hors des trois premières catégories. Nécessite une attention manuelle.

01

Init — Initialisation

Chargement de la configuration (moteur, clés API, signature). Vérification des paramètres obligatoires avant traitement.

02

Lecture — Parsing de l'email

Extraction de l'expéditeur, du sujet, du corps et des noms de pièces jointes fournis dans le formulaire.

03

Classification — Analyse du contenu

Moteur mots-clés : correspondance locale par dictionnaire. Moteur Claude : appel API Anthropic pour classification contextuelle.

04

Action — Génération de réponse

Création d'un brouillon de réponse adapté à la catégorie détectée, signé avec votre nom/signature configuré.

05

Journalisation — Enregistrement

Ajout de l'entrée dans le journal de session avec horodatage, catégorie, durée de traitement et résumé.

06

Clôture — Alertes Telegram (optionnel)

Si Telegram est configuré, une alerte est envoyée avec la catégorie et un résumé du traitement effectué.

04 — Configuration

Paramètres à configurer

Champ Requis Description
Moteur ✓ Oui Choisissez Mots-clés (local, sans clé) ou API Claude (nécessite une clé Anthropic).
Clé API Claude — Non Clé Anthropic au format sk-ant-.... Requise uniquement si le moteur Claude est sélectionné.
Nom / Signature ✓ Oui Nom ou titre dans les réponses générées. Ex : Dr. Grondin, MD
Bot Telegram Token — Non Token du bot Telegram obtenu via @BotFather. Format : 123456:ABC-DEF...
Telegram Chat ID — Non Identifiant numérique du chat destinataire des alertes automatiques.
AVIS

Votre clé API Anthropic reste dans votre navigateur et n'est jamais transmise à un tiers autre qu'Anthropic lors des appels API.

05 — Utilisation

Traiter un email pas à pas

1

Remplir les champs de l'email

Entrez l'expéditeur, le sujet, le corps du message et optionnellement les noms de pièces jointes (séparés par des virgules).

2

Vérifier la configuration

Assurez-vous que votre nom/signature est renseigné et que le moteur de classification souhaité est sélectionné.

3

Cliquer sur ▶ Exécuter le Workflow

Le pipeline démarre. L'état avance en temps réel dans la barre de workflow de l'interface.

4

Copier la réponse générée

La réponse s'affiche dans la zone de résultat. Utilisez ⎘ Copier la réponse pour la transférer dans votre client email.

5

Consulter le journal de session

Chaque traitement est consigné avec l'heure, la catégorie et un résumé. Utilisez ✕ Effacer le journal pour réinitialiser.

Exemple — Proof of Status

// Email déclenchant PROOF_OF_STATUS
Expéditeur : assurance@mutuelle-sante.ca
Sujet : Demande d'attestation médicale – dossier #2024-8821
Corps : Nous avons besoin d'une preuve de statut médical
         ainsi qu'une attestation signée par votre médecin.
06 — Telegram

Configurer les alertes Telegram

GRONDIN peut envoyer une notification automatique sur Telegram après chaque traitement. Voici comment configurer le bot :

1

Créer un bot avec @BotFather

Sur Telegram, cherchez @BotFather, tapez /newbot et suivez les instructions pour obtenir votre token.

2

Obtenir votre Chat ID

Envoyez un message à votre bot puis consultez https://api.telegram.org/bot<TOKEN>/getUpdates pour trouver votre chat.id.

3

Entrer les informations dans GRONDIN

Collez le token et le chat ID dans les champs correspondants. Cliquez sur Envoyer dans la section test pour vérifier la connexion.

INFO

Le bouton Envoyer dans "Test alerte Telegram" envoie un message de test immédiatement, sans traiter d'email.
