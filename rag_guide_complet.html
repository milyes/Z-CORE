Règles de découpage

  • 200–500 tokens par chunk
  • Overlap de 10–20%
  • Respecter les frontières sémantiques

02 — Retrieval

Retrieval hybride

Combiner la recherche sémantique et la recherche par mots-clés donne +15 à +30% de précision sur les datasets d'entreprise réels par rapport au vectoriel seul.

VECTORIEL

Similarité sémantique

Comprend le sens, les synonymes, le contexte implicite.

  • ✓ Requêtes en langage naturel
  • ✓ Multilingue
  • ✓ Synonymes, paraphrases
  • ✗ Codes exacts, IDs, acronymes
vs
KEYWORD (BM25)

Correspondance exacte

Parfait sur les termes précis, codes, noms propres.

  • ✓ Codes, références, IDs
  • ✓ Rapide, explicable
  • ✓ Noms propres, sigles
  • ✗ Sens, contexte, paraphrases

Reciprocal Rank Fusion (RRF)

Fusionne les deux listes de résultats en un seul score consolidé

score = 1/(k + rang_vectoriel) + 1/(k + rang_keyword)  — k = 60

Puis un cross-encoder reranker reclasse les top-20 résultats fusionnés (~50–150ms de latence additionnelle)

03 — Métriques

Framework RAGAS

RAGAS évalue automatiquement ton pipeline RAG à partir de 3 inputs : la question, le contexte récupéré, et la réponse générée.

F
Faithfulness — Fidélité
Chaque affirmation de la réponse est-elle traçable dans le contexte récupéré ? Un score bas signale que le LLM hallucine.
Seuil critique : > 0.70
AR
Answer Relevancy — Pertinence réponse
La réponse répond-elle bien à la question posée ? Une réponse peut être fidèle mais hors-sujet.
Seuil critique : > 0.65
CP
Context Precision — Précision contexte
Les chunks récupérés sont-ils tous utiles ? Un score bas indique trop de bruit dans le retrieval.
Seuil critique : > 0.60
CR
Context Recall — Rappel contexte
Toutes les informations nécessaires ont-elles été récupérées ? Un score bas = sources manquantes dans l'index.
Seuil critique : > 0.65

Table de diagnostic

Faithfulness Context Precision Context Recall Diagnostic Action recommandée
Bas Élevée Élevée LLM hallucine malgré de bons chunks Revoir le prompt système, contraindre la génération
Élevée Bas Moyen Retrieval ramène trop de bruit Améliorer chunking, ajouter reranker
Élevée Moyen Bas Sources manquantes dans l'index Étendre la base documentaire, re-indexer
Bas Bas Bas Problème à tous les niveaux Audit complet du pipeline
Élevée Élevée Élevée Pipeline sain Surveiller la dérive dans le temps

04 — Monitoring

Observabilité en production

L'observabilité répond à la question "que se passe-t-il ?" — mesurer, tracer, alerter sur 3 couches distinctes.

Modèle & performance
Latence p50/p99, tokens/seconde, taux d'erreur, coût par requête
TEMPS RÉEL
Qualité des outputs
Score RAGAS, détection d'hallucinations, dérive sémantique, feedback utilisateur
CRITIQUE
Données & contexte
Fraîcheur des documents, data drift, couverture de l'index, distribution des requêtes
BATCH
Boucle de correction
Alertes → Re-chunking → Ré-embedding → Revalidation automatique
AUTOMATISÉ
FRAÎCHEUR

Gestion du TTL

Chaque document indexé porte une date de création et une durée de vie (TTL). Un job batch vérifie quotidiennement les documents expirés et déclenche leur ré-indexation.

DATA DRIFT

Détection de dérive

Surveiller la distribution des embeddings dans le temps. Une dérive statistique (KL divergence > seuil) signale que les données d'usage s'éloignent de l'index.

ALERTE

Seuils d'alerte

  • Faithfulness < 0.70 → alerte critique
  • Latence p99 > 3s → alerte perf
  • Score similarité moyen < 0.55 → audit retrieval

05 — Gouvernance

Gouvernance & conformité

La gouvernance répond à "qui décide, qui est responsable, et selon quelles règles ?" — 4 piliers essentiels pour un système IA responsable.

01

Contrôle des accès

Qui peut interroger le système, avec quels droits, sur quelles bases documentaires. RBAC, audit des requêtes, isolation des données sensibles.

02

Traçabilité & audit

Log immuable de chaque décision : quelle question, quels chunks récupérés, quelle réponse produite. Nécessaire pour l'EU AI Act.

03

Éthique & biais

Détection automatique de biais dans les réponses, évaluation de l'équité sur différentes populations d'utilisateurs, red-teaming régulier.

04

Conformité réglementaire

RGPD (données personnelles), EU AI Act (classification du risque, supervision humaine obligatoire pour les systèmes à haut risque), normes sectorielles.

AVERTISSEMENT 2025

Une étude 2025 a montré que des systèmes RAG peuvent citer des sources exactes sans halluciner, mais rester pragmatiquement trompeurs — décontextualisant des faits, omettant des sources critiques. Pour les domaines critiques (médical, légal, finance), les métriques automatiques ne suffisent pas. Une supervision humaine reste nécessaire.
