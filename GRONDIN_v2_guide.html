01 — Aperçu

Qu'est-ce que GRONDIN v2 ?

GRONDIN Response Handler v2.0 est un outil web de traitement automatisé d'emails destiné aux professionnels de la santé. Il analyse le contenu d'un email entrant, le classe dans une catégorie, génère une réponse adaptée et peut envoyer une alerte via Telegram.

Deux moteurs de classification sont disponibles : mots-clés locaux (aucune connexion externe requise) ou API Claude (Anthropic) pour une classification plus intelligente et contextuelle.

INFO

L'outil fonctionne entièrement dans votre navigateur. Aucune donnée n'est envoyée à un serveur tiers, sauf si vous activez l'API Claude ou Telegram.

02 — Catégories de classification

Les 4 types de réponse

Chaque email traité est assigné à l'une des catégories suivantes :

🟢 POSITIVE_MEETING_REQUEST

Demande de rencontre ou de rendez-vous positif. L'email exprime une volonté de se rencontrer, une confirmation ou un suivi de réunion.

🟡 INFORMATIONAL_REQUIREMENT

Demande d'information générale. L'expéditeur cherche des renseignements, des documents ou des clarifications sur un sujet précis.

🔵 PROOF_OF_STATUS

Demande de preuve de statut ou de document officiel. Ex : lettre de statut, attestation, certificat médical, confirmation administrative.

🔴 ACTIF_QR_OTHER

Catégorie fourre-tout pour les emails qui ne correspondent pas aux trois catégories précédentes. Nécessite une attention manuelle.

Détail des étapes

01

INIT — Initialisation

Chargement de la configuration (moteur, clés API, signature). Vérification des paramètres obligatoires.

02

LECTURE — Parsing de l'email

Extraction de l'expéditeur, du sujet, du corps et des noms de pièces jointes fournis.

03

CLASSIFICATION — Analyse du contenu

Moteur mots-clés : correspondance locale par dictionnaire. Moteur Claude : envoi à l'API Anthropic pour classification contextuelle.

04

ACTION — Génération de la réponse

Création d'un brouillon de réponse adapté à la catégorie détectée, signé avec votre nom/signature configuré.

05

JOURNALISATION — Enregistrement

Ajout de l'entrée dans le journal de session avec horodatage, catégorie et résumé.

06

CLÔTURE — Envoi Telegram (optionnel)

Si Telegram est configuré, une alerte est envoyée avec la catégorie et un résumé du traitement.

04 — Configuration

Paramètres à configurer

Champ Obligatoire Description
Moteur ✓ Choisissez Mots-clés (local, aucune clé requise) ou API Claude (nécessite une clé Anthropic).
Clé API Claude — Clé API Anthropic au format sk-ant-.... Requise uniquement si le moteur Claude est sélectionné.
Votre nom / signature ✓ Nom ou titre qui apparaîtra dans les réponses générées. Ex : Dr. Grondin, MD
Bot Telegram Token — Token du bot Telegram (obtenu via @BotFather). Format : 123456:ABC-DEF...
Telegram Chat ID — Identifiant du chat ou canal où envoyer les alertes. Peut être un ID numérique ou un @username.
AVIS

Votre clé API Anthropic reste dans votre navigateur et n'est jamais transmise à un tiers autre qu'Anthropic lors des appels API.

05 — Utilisation pas à pas

Traiter un email

1

Remplir les champs de l'email

Entrez l'expéditeur, le sujet, le corps du message et optionnellement les noms des pièces jointes (séparés par des virgules).

2

Vérifier votre configuration

Assurez-vous que votre nom/signature est renseigné et que le moteur souhaité est sélectionné.

3

Cliquer sur ▶ EXÉCUTER LE WORKFLOW

Le pipeline démarre. L'état avance en temps réel dans la barre de workflow en bas de l'interface.

4

Lire et copier la réponse générée

La réponse s'affiche dans la zone de résultat. Utilisez le bouton ⎘ COPIER LA RÉPONSE pour la transférer dans votre client email.

5

Consulter le journal de session

Chaque traitement est consigné avec l'heure, la catégorie détectée et un résumé. Utilisez ✕ EFFACER LE JOURNAL pour réinitialiser.

Exemple de corps d'email — PROOF_OF_STATUS

// Email d'exemple déclenchant la catégorie PROOF_OF_STATUS

Expéditeur : assurance@mutuelle-sante.ca
Sujet : Demande d'attestation médicale – dossier #2024-8821
Corps : Bonjour, suite à votre demande de remboursement,
        nous avons besoin d'une preuve de statut médical
        ainsi qu'une attestation signée par votre médecin.
06 — Intégration Telegram

Configurer les alertes Telegram

GRONDIN peut envoyer une notification automatique sur Telegram après chaque traitement d'email. Voici comment configurer le bot :

1

Créer un bot avec @BotFather

Sur Telegram, cherchez @BotFather, tapez /newbot et suivez les instructions. Vous recevrez un token de type 123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11.

2

Obtenir votre Chat ID

Envoyez un message à votre bot, puis consultez https://api.telegram.org/bot<TOKEN>/getUpdates pour trouver votre chat.id.

3

Entrer les informations dans GRONDIN

Collez le token et le chat ID dans les champs correspondants. Cliquez sur ENVOYER dans la section test pour vérifier la connexion.

INFO

Le bouton ENVOYER dans la section "Test alerte Telegram" envoie un message de test immédiatement, sans traiter d'email.
